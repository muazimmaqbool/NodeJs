        The server verifies the signature and expiry ‚Äî if valid, it processes the request.

  4Ô∏è‚É£ Statelessness:
        JWTs are stateless.

        All the needed info is inside the token itself.
        As long as the server has the secret key (or public key) to verify the signature, it can validate any token without looking it up in a database.
        This makes JWTs easy to scale across multiple servers because there‚Äôs no shared session store.
